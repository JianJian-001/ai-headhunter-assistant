# 项目场景适配性分析

## 分析范围

本结论基于项目当前代码、技能定义与脚本能力进行静态分析，未包含线上真实数据、外部账号权限和第三方接口可用性的运行验证。

补充说明：

- 项目内未发现 `cursor_project_rules` 目录，因此本次分析依据现有仓库实现文件展开。
- 项目定位是“技能驱动的智能招聘平台”，核心能力围绕猎头业务流程构建，而不是通用内容平台。依据见 `README.md`、`src/lib/builtin-skills.ts`、`src/lib/skill-router.ts`、`src/lib/skill-runtime.ts`。

## 总体判断


| 场景       | 结论   | 判断                                      |
| -------- | ---- | --------------------------------------- |
| 岗位分析     | 明确支持 | 有独立技能与模板、输出流程、飞书落库流程                    |
| 公司分析     | 部分支持 | 可从公司维度抓取岗位信息，但未发现独立公司研究能力               |
| 面试辅导     | 部分支持 | 简历匹配链路包含面试辅助问答/面试建议，但不是独立面试辅导产品         |
| 薪酬调研     | 部分支持 | 多处使用薪资/薪酬字段和薪酬对标模板，但未见独立薪酬数据库或调研引擎      |
| 简历风险核查   | 明确支持 | 简历匹配技能直接输出简历可疑点/风险防范措施                  |
| 高端人才沟通辅助 | 明确支持 | 有首次触达、持续跟进、沟通节奏与渠道选择能力                  |
| 工作周报     | 部分支持 | 内置飞书工作流支持 standup/会议周报，但不属于主应用的专属猎头周报模块 |
| 猎头 IP 打造 | 暂不支持 | 未发现相关专用技能、页面、模板或工作流                     |


## 逐项分析

### 1. 岗位分析

结论：**明确支持**

依据：

- `src/lib/builtin-skills.ts` 注册了 `headhunter-search-report`，名称就是“岗位分析报告”。
- `skills/builtin/headhunter-search-report/SKILL.md` 明确要求生成“岗位深度分析报告”，并写入飞书文档和岗位库。
- `skills/builtin/headhunter-search-report/references/report-template.md` 包含岗位分析模板，且明确要求输出五维分析框架、薪酬对标、人才画像、成单难度等内容。

满足情况：

- 能围绕职位名称、公司、行业、职位描述生成结构化岗位分析报告。
- 能把报告沉淀到飞书文档和岗位库，适合猎头作业流转。

限制：

- 强依赖飞书链路与 `lark-cli`。
- 更偏“猎头岗位分析报告”，不是通用招聘咨询报告平台。

### 2. 公司分析

结论：**部分支持**

依据：

- `src/lib/builtin-skills.ts` 中 `headhunter-find-job` 的描述包含“看某公司职位列表”。
- `skills/builtin/headhunter-find-job/scripts/search_jobs.py` 提供 `company` 子命令，可搜索指定公司的岗位。
- `search_jobs.py` 输出字段包含公司、岗位、地点、薪资、链接。

满足情况：

- 能从招聘平台侧获取某公司的招聘岗位信息，用于侧面判断公司招聘活跃度、岗位分布、薪资带和扩张方向。

缺口：

- 未发现独立“公司分析”技能。
- 未发现公司组织架构、融资、业务阶段、竞争格局、雇主品牌、经营风险等研究模块。
- 目前更像“公司招聘情报收集”，不是完整“公司分析”。

### 3. 面试辅导

结论：**部分支持**

依据：

- `skills/builtin/headhunter-cv-jd-matching/SKILL.md` 明确写到脚本会生成“简历和职位匹配结果、简历可疑点、面试辅助问答”。
- `skills/builtin/headhunter-cv-jd-matching/scripts/main.js` 文件头写明能力包括“面试辅助问答”。
- `skills/builtin/headhunter-cv-jd-matching/scripts/main.js` 中存在“面试建议题目”输出区域。
- `skills/builtin/headhunter-candidate-report/references/role-setting.md` 提到“面试建议”。

满足情况：

- 能围绕某份简历和某个 JD 给出面试问题/建议题目。
- 能把面试建议放在候选人评估、推荐报告链路中辅助使用。

业务上更准确的目标应当是：

- 不是让客户主动提供可能会问的问题。
- 而是由猎头在电话沟通中沉淀“领导风格”“公司真实需求”“团队关注点”“过往进入面试候选人的反馈”。
- 再把这些经验和信息沉淀成面试辅导知识库，反向提供给候选人，提升面试成功率。
- 该能力的商业目标明确指向提升候选人面试通过率，从而提升猎头成单概率和回款效率。

缺口：

- 未发现独立“面试辅导”技能、页面或多轮模拟面试能力。
- 未发现按岗位题库、面试角色扮演、追问训练、答案点评这类专项辅导模块。
- 未发现“电话访谈纪要 -> 面试知识点提炼 -> 公司/面试官画像沉淀 -> 候选人面试辅导输出”的闭环。
- 未发现基于过往面试成功/失败案例的知识库维护机制。

如果按你的业务定义建设，这个能力更应该拆成以下模块：

1. 电话访谈信息沉淀：记录领导风格、团队风格、真实岗位诉求、雷点和隐性偏好。
2. 面试经验库：沉淀过往候选人面试反馈、常见追问、通过案例、失败案例。
3. 候选人辅导输出：按目标公司/岗位生成面试侧重点、沟通建议、回答框架和风险提醒。
4. 结果回流：把本次候选人的面试结果继续反哺知识库，形成可迭代闭环。

### 4. 薪酬调研

结论：**部分支持**

依据：

- `skills/builtin/headhunter-search-report/references/report-template.md` 明确要求“薪酬对标数据”和“市场薪酬区间”。
- `skills/builtin/headhunter-find-job/scripts/search_jobs.py` 可抓取岗位薪资字段。
- `skills/builtin/headhunter-candidate-sourcing/SKILL.md`、`references/ranking-routing.md`、`scripts/rank_candidates.py` 中均使用薪资/薪酬匹配逻辑。
- `skills/builtin/headhunter-table-manage/references/resources.md` 包含“薪资范围”“当前薪资”“期望薪资”等字段。

满足情况：

- 能在岗位分析、候选人匹配、候选人寻访中使用薪资信息做判断。
- 能做“客户预算 vs 市场区间”的粗颗粒度对标。

缺口：

- 未发现独立薪酬调研产品或标准化薪酬数据库。
- 未发现多维筛选后的行业/城市/职级薪酬统计分析模块。
- 当前更偏流程内“薪酬参考”和“预算匹配”，不等同于正式薪酬调研系统。

### 5. 简历风险核查

结论：**明确支持**

依据：

- `skills/builtin/headhunter-cv-jd-matching/SKILL.md` 明确要求输出“简历可疑点”和“风险防范措施”。
- `skills/builtin/headhunter-cv-jd-matching/scripts/main.js` 用于简历解析和匹配。
- `src/app/api/chat/route.ts` 支持附件上传与提取文本，能够把简历文件带入对话链路。
- `README.md` 明确写到支持“附件支持：上传图片、文档等附件参与对话，支持内容提取与解析”。

满足情况：

- 能对单份简历或简历文件夹进行解析、匹配和风险提示。
- 能识别简历疑点，适合做初筛风控。

业务上更准确的目标应当是：

- 不是把“风险核查”停留在一次性 AI 判断。
- 而是维护一套持续演进的“简历可疑点库”和“快速判断规则库”。
- 新简历进入后，先按规则快速扫描可疑项，再把风险点交给猎头在后续面试中核验。
- 这个能力的核心价值是“提前提醒风险项 + 辅助后续面试核验”，而不是直接替代背调。

限制：

- 这里的“风险核查”更偏简历真实性/匹配性疑点识别，不等同于正式背调系统。
- 依赖外部简历解析接口与脚本执行环境。
- 当前仓库没有看到独立维护的“简历 bug 库”“风险规则库”“可疑点分类体系”。
- 当前也没有看到专门页面承载风险分级、证据定位、核验问题建议和规则维护。

如果按关键能力建设，建议补齐以下能力：

1. 简历可疑点库：频繁跳槽、时间重叠、学历时间异常、岗位级别虚高、成果表述失真、技术栈堆砌、项目描述空泛等。
2. 规则引擎：基于时间、关键词、经历连续性、岗位跨度、薪资跨度等规则快速扫描。
3. 风险标签体系：按高/中/低风险分级，并区分真实性风险、稳定性风险、能力夸大风险、匹配风险。
4. 面试核验建议：针对每个风险点自动生成后续面试追问问题。
5. 风险核查页面：展示风险总分、风险标签、证据片段、建议追问、核验结论和规则维护入口。

### 6. 高端人才沟通辅助

结论：**明确支持**

依据：

- `src/lib/builtin-skills.ts` 中有 `headhunter-outreach-message` 和 `headhunter-greeting-skill` 两个直接面向候选人沟通的技能。
- `skills/builtin/headhunter-outreach-message/SKILL.md` 支持首触渠道选择、首触文案、无回复跟进、触达节奏建议。
- `skills/builtin/headhunter-greeting-skill/SKILL.md` 面向已建联后的持续跟进和节点保温。
- `skills/builtin/headhunter_shared/python/headhunter_shared/messaging_runtime.py` 体现了多阶段跟进逻辑，并包含团队亮点、业务背景、薪酬信息等沟通焦点。

满足情况：

- 适合猎头在高端候选人首触、跟进、保温阶段提升沟通质量。
- 已覆盖电话、微信、短信、邮件、脉脉、领英等渠道建议。

限制：

- 侧重“沟通策略与文案辅助”，不是自动化 CRM 外呼系统。
- 未直接看到企业微信/短信网关等发送执行闭环。

### 7. 工作周报

结论：**部分支持**

依据：

- `skills/builtin/lark-cli/cli/skills/lark-workflow-standup-report/SKILL.md` 支持把日程和待办汇总成摘要。
- `skills/builtin/lark-cli/cli/skills/lark-workflow-meeting-summary/SKILL.md` 描述中明确支持“会议周报”。
- `README.md` 明确项目内含飞书集成技能。

满足情况：

- 如果用户工作流在飞书中，项目具备把日程、待办、会议纪要整理成摘要/周报的能力。

缺口：

- 该能力主要来自 `lark-cli` 技能集，不是主应用首页中的专用业务模块。
- 未发现“猎头顾问工作周报”专属模板，例如本周新增岗位数、推荐数、面试数、offer 数、回款预测等经营指标周报。

### 8. 猎头 IP 打造

结论：**暂不支持**

依据：

- `src/lib/builtin-skills.ts` 当前内置技能集中在招聘流程、候选人管理、飞书工作台，不包含个人品牌/内容运营类技能。
- 仓库内检索“猎头IP”“个人IP”“内容创作”“品牌打造”“公众号”“小红书”等关键词，未发现相关技能、模板或页面实现。

判断说明：

- 首页和通用聊天能力理论上可以让模型生成文案，但这属于通用大模型能力，不属于项目内已经产品化的场景支持。
- 按“是否能够满足场景需求”的标准，当前仓库没有足够证据证明它已经支持“猎头 IP 打造”。

## 结论建议

如果只看当前仓库可证明的能力，这个项目更适合定位为：

1. 猎头流程型 AI 助手
2. 招聘岗位分析与候选人寻访/匹配/沟通平台
3. 带飞书工作台与文档沉淀能力的招聘协作工具

如果按你补充的业务目标重新排序，最优先补齐的方向应调整为：

1. 面试辅导专用技能：围绕电话访谈信息、历史面试反馈和成功案例建立知识库，直接提升候选人面试通过率。
2. 简历风险核查页面：围绕可疑点库、规则库、证据定位和面试核验建议做成强交互页面。
3. 猎头 IP 打造能力：内容选题、平台分发、个人品牌定位、话术/文章生成模板。
4. 公司分析专用技能：公司画像、业务阶段、赛道分析、组织与招聘策略。
5. 薪酬调研专用能力：标准化薪酬数据、横向对比和分层统计。

其中前两项更接近猎头核心收入链路：

1. 面试辅导决定候选人能否顺利通过客户面试。
2. 简历风险核查决定候选人是否值得继续投入推进。
