# Workflow 详情

## 流程概览

确定主题+候选人 → 生成预览 → 发送 → 追加明细

> 加载后先判断主题，生成问候和预览文件。用户确认发送后才检查微信并发送。

## 确定主题+候选人

判断主题：
- 用户指定了节日 → 直接用
- 用户说"日常关怀" → 主题定为日常关怀
- 用户未指定 → web_search 搜索今天节日

获取候选人：
- 触发消息中已包含姓名 → 直接用
- 否则问"请提供候选人姓名或微信号"

## 生成预览

**此步不碰微信、不调用脚本命令**。

1. 先读 `references/notes.md`
2. 生成问候内容
3. 创建 `greeting/YYYY-MM-DD-001.md`
4. 展示预览表格和文件链接
5. 等用户回复"确认发送"

## 发送

**用户确认后**才执行：

```text
提示：> **⚠️ 即将操作微信，请暂时不要移动鼠标和键盘！发送完成后会自动隐藏微信**
重新读取预览文件
逐条执行 send：
  ① 检查微信已安装+已登录
  ② 搜索联系人
  ③ 发送消息
  ④ 关闭聊天并隐藏微信
```

## 追加明细

每发送完一位候选人，立即把结果追加到预览文件。

补发/追发流程与首次发送完全一致。
