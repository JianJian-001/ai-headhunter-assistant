# 工作流程详情

## 数据提取策略

| 优先级 | 方法 | 适用场景 |
|--------|------|---------|
| 1 | browser_evaluate | 快速提取链接，Shadow DOM 页面可能失败 |
| 2 | browser_snapshot | 获取交互元素（搜索框、筛选器） |
| 3 | browser_screenshot + 图像识别 | 最可靠，用于提取岗位列表内容 |

**关键**: evaluate 返回空数组时，立即切换截图方案

## 详细步骤

### 1. 场景判断

记录参数：公司名、岗位关键词、城市、校招/社招

### 2. 确定URL

**搜索模板**:
- BOSS直聘: `https://www.zhipin.com/web/geek/job?query={关键词}`
- 猎聘: `https://www.liepin.com/zhaopin/?key={关键词}`

公司官网参考 [official-hr-site.md](./official-hr-site.md)

### 3. 导航并等待

```
browser_navigate(url)
browser_wait_for(timeMs=2500)
```

### 4. 登录检测

```
browser_screenshot(fullPage=false)
检查: "登录"、"扫码"、"验证手机"、"请先登录"
```

有登录墙 → 提示用户登录后继续

### 5. 搜索筛选

```
1. snapshot() → 找筛选器/搜索框
2. click筛选器 → wait_for(1500)
3. 或: click搜索框 → type关键词 → press enter → wait_for(2000)
```

### 6. 获取岗位列表

```
LOOP (最多2页):
  1. wait_for(1000)
  2. 尝试 evaluate 提取链接 → 空则切换截图
  3. screenshot(fullPage=true) → 图像识别提取：
     - 岗位名称、公司/部门、地点、薪资、岗位链接
  4. 判断到底: "没有更多"/连续相同/达最大页数
  5. 翻页: click下一页 或 scroll_down
```

**岗位链接策略**: JS提取 → 点击详情页获取URL → 构造URL

### 7. 整理结果

**文件**: `jobs/YYYY-MM-DD-{标识}-001.md`

```markdown
## 岗位列表

获取时间：YYYY-MM-DD HH:MM:SS

### 来源：{名称}
数据来源：[来源](URL)

| # | 岗位名称 | 公司/部门 | 工作地点 | 薪资 | 详情链接 |
|---|---------|----------|---------|------|---------|
| 1 | ... | ... | ... | ... | [查看](链接) |
```

**对话输出**:

```markdown
## {标题} 相关岗位

共从 **N 个来源** 找到 **X 个岗位**。

### {来源}（N 个）
| 岗位名称 | 公司/部门 | 地点 | 薪资 | 详情 |
|---------|----------|------|------|------|
| ... | ... | ... | ... | [查看](链接) |

岗位列表已保存: [jobs/...](file://路径)
```

## 常见问题

| 问题 | 解决 |
|------|------|
| evaluate 返回空 | 切换截图方案 |
| snapshot 内容少 | 增加等待时间 |
| 找不到筛选器 | 用搜索框输入关键词 |
| 岗位链接无法提取 | 点击进入详情页获取 |
| 搜索无结果 | 提示更换关键词 |
| 某来源失败 | 跳过，继续其他来源 |
