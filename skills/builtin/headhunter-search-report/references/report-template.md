# 猎头岗位需求深度分析报告模板

> 本模板定义了岗位解析报告的完整结构和样式规范。生成报告时，必须严格遵循五维分析框架的递进式结构，以及样式和质量要求。
>
> **核心原则**：
> - 五个维度的框架和顺序必须保留，不可删除或调换
> - 每个维度必须包含「猎头洞察」（一句话概括核心发现）和「分析结论」
> - 维度三必须包含需求优先级拆解表格
> - 维度四必须包含薪酬对标数据和人才画像
> - 维度五必须给出明确的接单决策建议
> - 每个维度内部的具体分析内容可以根据行业特点灵活选择、拓展或补充
> - 报告中禁止使用 emoji 或图标


## 报告标题格式（连字符'-'前后禁止有空格）

```
{公司名称}-{职位名称}-岗位解析报告
```

## 维度一：商业背景与战略定位分析

**猎头洞察：[一句话概括核心发现]**

理解企业为什么在此时此刻需要这个岗位，判断岗位的战略本质。

**分析要点**：

- **企业发展阶段**：分析公司所处阶段（初创期/高速增长期/成熟期/转型期），以及近期业务动态、融资情况、行业竞争格局
- **招聘战略意图**：该岗位是新市场拓展、新产品研发、内部优化还是降本增效？是新增编制（扩张）、替补（前任离职/晋升/淘汰）还是项目制？
- **岗位价值链定位**：该岗位属于利润中心（直接创收）还是成本中心（支撑保障）？这决定了候选人需要具备的核心思维模式
- **负面信息排查**：是否存在裁员、欠薪、诉讼、资金链危机等负面新闻，这些直接影响候选人的入职意愿

**分析结论：**

[总结该岗位的战略本质，明确候选人需要具备什么类型的心态（如开拓型先锋 vs 稳态管理者），以及这对寻访策略的影响]

## 维度二：组织架构与团队生态分析

**猎头洞察：[一句话概括核心发现]**

分析候选人将进入的组织生态，评估文化适配和人际关系要求。

**分析要点**：

- **汇报关系**：该岗位向谁汇报（实线/虚线），汇报对象的管理风格如何
- **团队规模与构成**：下属团队人数、团队成员背景、团队当前状态
- **跨部门协作**：需要与哪些关键部门协同，协作中的常见摩擦点
- **企业文化**：真实的企业文化（而非官方宣传），是狼性文化、流程导向、创新驱动还是层级分明

**分析结论：**

[总结候选人需要具备的组织适应能力和人际特质，明确什么样的人能在这个生态中生存和发展]

## 维度五：操作可行性与商业价值评估

**猎头洞察：[一句话概括核心发现]**

从猎头公司经营角度，评估该项目是否值得投入资源。

**分析要点**：

- **成单难度**：综合薪酬竞争力、人才稀缺度、企业品牌吸引力，评估成单概率
- **沟通成本**：是否需要大量时间进行"市场教育"（说服客户调整预算或用人标准）
- **商业回报**：按猎头费率估算单笔服务费，与投入的精力对比，评估投入产出比
- **客户配合度**：面试流程效率、反馈速度、内部决策一致性

**最终决策建议：**

| 优先级 | 判断标准 | 建议行动 |
|:------|:------|:------|
| **优先级甲（全力投入）** | 高紧迫度 + 有竞争力的薪酬 + 预期一致 + 资源匹配 | 全团队投入，配备专属寻访 |
| **优先级乙（机会型跟进）** | 信号中等，存在风险但可控 | 机会型寻访，利用现有人才库 |
| **建议婉拒** | 低紧迫度 + 不切实际的预期 + 低回报 + 资源不匹配 | 礼貌婉拒，或先进行市场教育 |

## 质量要求（生成报告时必须遵守）

1. **内容详实**：每个维度的内容必须有深度和细节，使用完整段落而非简单罗列，禁止一两句话敷衍了事，禁止空洞的套话
2. **数据支撑**：分析和判断必须有数据或事实支撑，引用薪酬报告、行业数据等来源
3. **表格优先**：需求优先级拆解、目标公司地图、决策建议**必须以表格形式输出**，其他维度在适合时也应使用表格呈现结构化信息
4. **真实性**：目标公司必须是真实存在的知名企业，禁止编造
5. **市场性**：薪酬数据符合当前市场行情，推断需说明依据
6. **可操作性**：所有分析和建议必须具有可操作性，猎头拿到报告就能直接执行
7. **针对性**：内容紧密围绕用户提供的具体职位信息，禁止泛泛而谈
8. **专业性**：使用猎头行业专业术语，体现行业洞察和专业深度
9. **有理有据**：推荐和判断必须说明理由，让猎头理解"为什么"
10. **简洁专业**：报告风格简洁专业，善用加粗、要点列表、表格等提升可读性，避免过度装饰
