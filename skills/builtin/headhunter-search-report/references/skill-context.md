# 岗位解析报告 — 补充参考

## 五维分析框架

报告严格遵循五维分析框架，内容详实、简洁专业。生成报告前**必须先读取** `report-template.md` 模板，严格按照模板的结构、样式和质量要求生成。

- **维度一：商业背景与战略定位分析** — 理解企业为什么需要这个岗位，判断岗位的战略本质
- **维度二：组织架构与团队生态分析** — 分析候选人将进入的组织生态，评估文化适配要求
- **维度三：岗位硬性与软性需求拆解** — 按优先级拆解岗位需求（必须项/应该有/可以有/隐性需求）
- **维度四：市场对标与人才画像绘制** — 评估薪酬竞争力、人才池深度，绘制精准人才画像
- **维度五：操作可行性与商业价值评估** — 从猎头经营角度评估项目是否值得投入

五个维度的框架和顺序必须保留，每个维度内部的具体分析内容可以根据行业特点灵活选择、拓展或补充。

### 报告生成要求

- 报告标题格式：{公司名称}-{职位名称}-岗位解析报告
- 报告结构：案例背景 → 维度一至维度五 → 总结
- 每个维度必须包含「猎头洞察」和「分析结论」
- 维度三必须包含需求优先级拆解表格（必须项/应该有/可以有/隐性需求）
- 维度四必须包含目标公司地图表格
- 维度五必须包含接单决策建议表格
- 每个维度内容必须详实，使用完整段落，禁止一两句话敷衍
- 风格简洁专业，善用加粗、要点列表、表格等提升可读性

## 岗位库字段映射

岗位信息通过 `headhunter-table-manage` 技能写入「一人猎头数据站」的「岗位库」数据表。

| 岗位库字段 | 类型 | 数据来源 |
|------|------|---------|
| 岗位名称 | text | 用户提供的职位名称 |
| 所属公司 | text | 用户提供的公司名称 |
| 岗位类型 | singleSelect | 根据岗位性质判断：技术岗/销售岗/职能岗 |
| 岗位解析 | text | 报告概括（150 字以内的一段话） |
| 薪资范围 | text | 报告第一部分第 3 维度的薪酬预估 |
| 招聘状态 | singleSelect | 默认填"进行中" |
| 岗位链接 | url | 步骤 5 生成的飞书在线文档 URL |
| 备注 | text | 寻访关键字（从报告搜索策略中提取，逗号分隔） |

## 写入前格式检查规则

在写入飞书文档前，必须对本地 Markdown 文件执行以下检查，发现问题立即修复后再写入：

### 检查一：引号转义

- 扫描文件中所有英文双引号 "，确认每处双引号都是成对出现的语义引号（如中文引用、术语标注）
- 若存在单独出现、未闭合的英文双引号，将其替换为中文引号（“ ”）或删除，避免 shell 解析时截断内容

### 检查二：Markdown 格式规范性

- 每个标题行（# / ## / ###）的前后必须各有一个空行，否则渲染时会将紧跟其后的内容误判为标题的一部分，导致全文样式错乱
- 列表项（- / * / 数字序号）前后必须各有一个空行，列表项之间不能有多余空行
- 表格前后必须各有一个空行，表格的分隔行（|---|---|）必须存在且列数与表头一致
- 段落之间必须用空行分隔，不能只用换行符（单个换行在 Markdown 中不会产生段落分隔）

### 检查三：特殊字符

- 扫描文件中是否存在未转义的 $ 符号（非变量引用场景），若有需替换为 \$，避免 shell 变量展开破坏内容
