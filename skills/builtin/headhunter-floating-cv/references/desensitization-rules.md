# 脱敏规则说明

## 目标

在不改变候选人核心经历真实性的前提下，隐藏可直接识别身份或过度暴露客户资源的信息。

## 常见脱敏对象

1. 姓名：可替换为“候选人 A”或姓氏缩写。
2. 手机号、邮箱、微信号：默认全部隐藏。
3. 身份证号、家庭住址、生日等个人敏感信息：默认全部隐藏。
4. 现公司/目标公司中的敏感客户名、项目代号、系统名：按用户约定替换为泛化描述。
5. 学校、公司名称、产品名称：只有在用户要求时才做泛化；不得擅自处理。

## 脱敏动作

1. `隐藏`：直接替换为 `[已脱敏]`。
2. `泛化`：改写为行业、级别、公司类型、业务方向等抽象描述。
3. `保留`：按原文保留，不改写。

## 输出原则

1. 脱敏后应保持时间线、职级、职责和成果逻辑可读。
2. 不因脱敏而新增不存在的项目背景或业务结果。
3. 同一类字段在全文中保持一致的替换口径。
4. 用户未明确说明时，优先隐藏直接识别个人身份的信息，其他字段先追问。
