
生成报告时按照以下结构输出 Markdown 内容。每位候选人独立生成一份报告，**总字数控制在 300 字以内**。


## 生成规则 **需要严格执行**

1. **总字数 ≤ 300 字**：只保留最关键的信息，语言精炼
2. **一人一报告**：每位候选人独立生成一份报告
3. **数据驱动**：所有评价必须基于飞书妙记会议记录内容和简历相关信息，不要编造
4. **考核维度精炼**：从 JD 中提炼 3-5 个核心维度，每个维度用一句话评价
5. **字段缺失处理**：未提及的信息直接省略，不要编造
6. **Markdown 兼容**：输出需兼容飞书文档渲染
7. **报告内容不要插入emoji**：输出不要插入表情符号
8. **内容要求**：首行不需要类似于"xx推荐报告"名称字样，标题会有
9. **标题格式**：一、二、三、四等标题请使用二级标题

```markdown

> 报告日期：xxxx年xx月xx日
> 分析师：悟空

## 一、基本信息

| 维度 | 内容 |
|------|------|
| 姓名 | <候选人姓名> |
| 当前公司/职位 | <公司> · <职位>（如有） |
| 工作年限 | <工作年限>（如有） |
| 学历/院校 | <学历>（如有） |

## 二、核心优势总结

用 2-3 句话概括候选人最突出的竞争力，说明为什么值得关注。

## 三、JD 考核维度匹配分析

从 JD 中提炼 3-5 个核心考核维度，逐一评估匹配程度，采用五分评级制，以表格形式展示。

| 考核维度 | 评分 | 匹配说明 |
|---------|------|---------|
| <维度1> | 4/5 | <一句话评价> |
| <维度2> | 3/5 | <一句话评价> |
| ... | ... | ... |
| **综合匹配度** | **X.X/5** | |

## 四、推荐理由

**推荐结论**：推荐 / 不推荐
**理由**：< 站在猎头视角，用 3-5 句话说明推荐或不推荐该候选人的核心理由，结合匹配分析给出明确结论，理由中不要给出“唯一”，“最”等绝对字样。>
```
