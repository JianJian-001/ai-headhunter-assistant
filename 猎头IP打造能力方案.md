# 猎头 IP 打造能力方案

## 一、目标定义

这里的“猎头 IP 打造能力”不是简单帮用户写几篇文案，而是帮助猎头持续建立一个可被候选人、客户和行业认知的专业形象。

这个能力的核心目标是：

1. 让猎头在细分赛道上形成稳定的人设和专业标签。
2. 让候选人愿意主动添加、回复、信任和长期保持联系。
3. 让客户认为这个猎头“懂行业、懂岗位、懂人才”。
4. 让内容生产形成持续流量入口，而不是一次性发文。
5. 最终服务于线索获取、候选人转化、客户转化和成单。

## 二、猎头 IP 的本质

猎头 IP 的本质不是“会写内容”，而是把猎头在一线沟通中积累的真实洞察，转化成可持续输出的专业内容资产。

对猎头最有价值的 IP，一般来自以下几类真实信息：

1. 客户电话里听到的真实需求。
2. 候选人沟通中暴露出的跳槽动机和顾虑。
3. 面试推进中观察到的公司风格、团队偏好和决策逻辑。
4. 岗位分析、人才地图、薪酬差异、招聘趋势等一线认知。
5. 候选人常问问题和常见误区。

所以，IP 打造能力的核心，不是从零编内容，而是把这些一线信息沉淀、提炼、改写、分发。

## 三、这个项目里应该建设什么能力

如果这个项目要支持“猎头 IP 打造”，建议把能力拆成 6 个模块。

### 1. 人设定位模块

用于回答“这个猎头要让别人记住什么”。

需要支持的内容：

- 细分赛道定位：比如 AI、智能制造、消费互联网、跨境电商、医疗器械。
- 服务对象定位：偏候选人、偏客户、偏创业公司创始人、偏中高端管理岗。
- 专业标签定位：懂组织搭建、懂薪酬谈判、懂面试辅导、懂高端人才沟通。
- 表达风格定位：专业型、直白型、行业观察型、顾问型、故事型。

输出结果建议包括：

- 一句话人设
- 三个核心标签
- 内容边界
- 不建议触碰的话题

### 2. 内容选题引擎

用于解决“每天写什么”。

选题来源不应靠灵感，而应来自猎头日常工作流。

建议接入的选题来源：

1. 岗位分析报告
2. 面试辅导知识库
3. 简历风险核查结果
4. 候选人沟通高频问题
5. 客户需求变化记录
6. 薪酬与市场观察
7. 成功案例与失败复盘

建议输出的选题类型：

- 行业观察类
- 求职避坑类
- 面试辅导类
- 简历诊断类
- 高端人才沟通类
- 客户招聘洞察类
- 成交案例复盘类

### 3. 内容生成与改写模块

用于解决“同一个洞察如何变成不同平台能发的内容”。

建议支持：

1. 长文输出：适合公众号、知识库文章、朋友圈长文。
2. 短帖输出：适合小红书、脉脉、LinkedIn、朋友圈短内容。
3. 标题改写：同一主题生成多个标题版本。
4. 开头钩子：增强阅读和转发意愿。
5. 平台适配：按公众号、视频号、小红书、脉脉、LinkedIn 生成不同语气。
6. 私聊转化话术：内容发布后，如何承接私信和进一步沟通。

这里的重点不是“写得像 AI”，而是“写得像一个真实做过单、有行业认知的猎头”。

### 4. 内容资产库

用于解决“内容积累以后如何复用”。

建议维护以下内容库：

1. 选题库
2. 爆款标题库
3. 案例库
4. 高质量观点库
5. 候选人常见问题库
6. 客户高频异议库
7. 平台模板库

每条内容资产建议都带以下字段：

- 主题
- 赛道
- 面向对象
- 使用场景
- 内容类型
- 来源证据
- 是否已发布
- 发布平台
- 数据表现

### 5. 分发与发布节奏模块

用于解决“不是写完就结束，而是持续运营”。

建议支持：

1. 周内容计划
2. 多平台分发建议
3. 一稿多发改写
4. 发布时间建议
5. 连载主题策划
6. 互动跟进提醒

推荐的最小节奏可以是：

- 每周 1 篇深度内容
- 每周 3 到 5 条短内容
- 每周 1 次案例复盘
- 每周整理一次评论、私信和潜在线索

### 6. 转化承接模块

这是最关键的一步。

猎头 IP 的目标不是单纯做曝光，而是产生业务结果。

因此建议把内容和转化打通：

1. 内容后续私聊话术
2. 候选人咨询接待脚本
3. 客户咨询接待脚本
4. 高意向线索标记
5. 跟进提醒
6. 从内容线索转到顾问沟通

## 四、适合猎头的 IP 内容方向

从业务价值来看，猎头最适合做的内容方向有 5 类。

### 1. 岗位与行业洞察

例如：

- 为什么最近 AI 产品经理岗位越来越难招
- 某类岗位为什么总是卡在终面
- 某行业公司招人时最看重什么

### 2. 面试辅导

例如：

- 某类公司领导最关注的候选人特质
- 候选人在面试里最容易踩的坑
- 面试前应该如何理解真实岗位诉求

### 3. 简历风险核查

例如：

- 哪些简历写法最容易让 HR 或业务方起疑
- 简历里哪些时间线问题最危险
- 为什么“看起来很漂亮”的简历反而容易挂

### 4. 候选人沟通与职业决策

例如：

- 高端候选人为什么不愿意回猎头消息
- 候选人接到机会后最看重什么
- 如何和候选人聊机会而不让对方有防备感

### 5. 成交案例与复盘

例如：

- 某次推荐为什么成功
- 某次 offer 为什么失败
- 某个高潜候选人为什么最后没入职

## 五、这个能力应该如何接入现有项目

结合当前项目结构，建议不要把“猎头 IP 打造”只做成一个通用聊天 prompt，而是做成独立技能或页面能力。

推荐接入方式：

### 方案 A：先做独立技能

新增一个类似 `headhunter-ip-builder` 的技能，职责包括：

1. 提取用户提供的一线信息
2. 判断适合的人设方向
3. 输出选题建议
4. 生成多平台内容草稿
5. 给出发布和转化建议

适合第一阶段快速验证。

### 方案 B：再做独立页面

当积累一定使用量后，建议升级成页面化能力，至少包括：

1. 人设配置区
2. 选题管理区
3. 内容生成区
4. 内容资产库
5. 发布计划区
6. 线索转化区

如果只靠聊天窗口，这个能力会很快碎片化，难以沉淀资产。

## 六、知识库建设建议

猎头 IP 打造要真正可用，必须依赖知识库，而不是只靠模型临场发挥。

建议重点维护以下知识库：

### 1. 行业洞察库

- 赛道趋势
- 高发岗位
- 常见招聘难点
- 人才供需变化

### 2. 面试经验库

- 领导风格
- 公司真实需求
- 常见追问
- 面试通过案例
- 面试失败案例

### 3. 简历风险库

- 常见简历 bug
- 高风险表述
- 时间线异常规则
- 能力夸大规则

### 4. 候选人沟通素材库

- 高频顾虑
- 高频问题
- 成功沟通案例
- 拒绝后的二次触达案例

### 5. 内容模板库

- 公众号模板
- 小红书模板
- 脉脉模板
- LinkedIn 模板
- 朋友圈模板

## 七、能力输出示例

一个成熟的“猎头 IP 打造能力”至少应输出以下结果：

1. 人设建议：你适合做“AI 行业中高端岗位猎头顾问”。
2. 本周选题：给出 10 个围绕真实业务的内容选题。
3. 平台改写：同一选题生成公众号版、小红书版、脉脉版。
4. 发布计划：本周一到周五每天发什么。
5. 转化建议：每条内容后如何承接私信和线索。
6. 数据复盘：哪些内容带来咨询，哪些内容只有曝光没有转化。

## 八、衡量这个能力是否有效

不能只看“写出来了多少内容”，而要看业务指标。

建议跟踪：

1. 每周产出内容数
2. 每周发布次数
3. 阅读、收藏、点赞、转发
4. 私信咨询量
5. 新增候选人线索数
6. 新增客户线索数
7. 从内容进入沟通的人数
8. 最终转化为面试、offer、入职的数量

## 九、当前项目的结论

基于当前仓库，项目**暂不具备完整的猎头 IP 打造能力**。

当前缺口主要有：

1. 没有人设定位模块
2. 没有内容选题引擎
3. 没有内容资产库
4. 没有平台化改写能力设计
5. 没有发布计划与节奏管理
6. 没有内容到业务转化的承接链路

但这个项目已经有几个适合作为 IP 内容源头的能力基础：

1. 岗位分析
2. 面试辅导
3. 简历风险核查
4. 高端人才沟通辅助

也就是说，现有项目可以作为“内容素材生产底座”，但还不是“IP 运营系统”。

## 十、优先建设建议

如果要落地，建议按以下顺序推进：

1. 先做 `猎头IP打造技能`
   目标：先验证内容生成、选题生成、平台改写是否有真实需求。

2. 再做 `内容资产库`
   目标：把岗位分析、面试辅导、简历风险核查沉淀成可复用内容素材。

3. 再做 `发布计划与转化承接`
   目标：让内容和业务结果连接起来。

4. 最后做 `独立页面`
   目标：形成完整的猎头 IP 运营工作台。

## 十一、一句话结论

猎头 IP 打造能力，本质上是把猎头在客户、候选人和岗位中的一线洞察，沉淀成可持续输出、可多平台分发、可带来线索转化的内容系统。

对于当前项目来说，这是一项值得优先补齐的增长型能力，但必须做成“知识库 + 内容引擎 + 资产库 + 转化链路”的组合，而不能只做成一个写文案的聊天功能。
