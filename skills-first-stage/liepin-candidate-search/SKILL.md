# 【猎聘平台专属】候选人搜索 Skill

## 🚨 排他性规则（必须严格遵守，避免和其他平台冲突）

|------|------|
| ✅ 可使用场景 | 用户明确提到"猎聘" / 明确要求"在猎聘上搜索" / 明确指定猎聘平台 |
| ❌ 绝对禁止场景 | 1. 用户提到其他招聘平台：BOSS直聘、脉脉、智联招聘、前程无忧、拉勾网、BOSS、直聘等<br>2. 用户泛泛说"找人"、"找候选人"没有指定平台<br>3. 任何非猎聘平台的搜索请求 |

## 功能说明

此 skill 为猎聘平台专属，用于在猎聘网（liepin.com）搜索候选人并导出数据，支持以下特性：
- ✅ **自然语言识别**：自动从用户输入中提取岗位关键词、数量、城市、工作年限、学历等筛选条件
- ✅ **智能参数确认**：提取参数后自动展示给用户确认，避免搜索错误
- ✅ **语义化文件名**：强制带"猎聘_"前缀，根据查询条件自动生成文件名，包含所有筛选维度，方便和其他平台的结果区分
- ✅ **自动防风控**：内置翻页间隔、超时保护，避免猎聘账号被限制
- ✅ **Excel兼容**：自动添加UTF-8 BOM头，打开无乱码
- ✅ **结果校验**：自动统计抓取数量，生成数据预览

在开始操作前，**必须**按顺序检查以下项目：


## 使用流程

### 步骤 0: 平台判断（必须第一步执行，避免平台冲突）
1. 检查用户输入是否明确提到"猎聘"，或者明确指定在猎聘平台搜索
2. 如果用户提到其他平台名称，直接终止使用此skill
3. 如果用户没有指定平台，先询问："请问你是要在猎聘平台上搜索吗？"，用户确认是猎聘后再继续

---

### 步骤 1: 智能参数提取
自动从用户的自然语言输入中提取以下参数：
| 参数 | 提取规则 | 默认值 |
|------|----------|--------|
| 搜索关键词 | 岗位名称（如"AI产品经理"、"Java开发"、"前端"） | **必填，从用户输入中提取** |
| 抓取数量 | 数字+单位（如"100个"、"200位"、"50人"） | 100（最大支持1500） |
| 所在城市 | 城市名称（如"上海"、"北京"、"杭州"） | 空（全国） |
| 期望城市 | 关键词"期望到XX"、"想去XX工作" | 空（不筛选） |
| 工作年限 | "X-Y年"、"X年以上"、"应届"、"3-5年" | 空 |
| 学历要求 | "本科"、"硕士"、"博士"、"大专" | 空 |

**提取示例：**
用户输入："帮我找200个上海的3-5年工作经验的本科Java开发"
自动提取结果：
- 关键词：Java开发
- 数量：200
- 城市：上海
- 工作年限：3-5年
- 学历：本科

用户确认后再执行下一步，如有调整则修改对应参数。

### 步骤 3: 智能生成文件名（强制带"猎聘_"前缀，避免和其他平台结果冲突）
根据确认后的参数自动生成语义化文件名，**必须以"猎聘_"开头**，规则：
```
猎聘_{城市}_{工作年限}_{学历}_{关键词}_候选人_{数量}位_{当前日期}.csv
```
- 强制"猎聘_"前缀，从文件名就能明确区分是猎聘平台的数据，和其他平台（BOSS、脉脉等）的结果不混淆
- 没有的筛选条件自动省略
- 多个筛选条件按顺序排列
- 文件名清晰展示搜索维度，方便后续管理

**示例文件名（均带猎聘前缀）：**
1. 无筛选条件：`猎聘_AI产品经理_候选人_100位_20260415.csv`
2. 带城市：`猎聘_北京_产品经理_候选人_150位_20260415.csv`
3. 多维度筛选：`猎聘_上海_3-5年_本科_前端开发_候选人_200位_20260415.csv`

### 步骤 4: 执行搜索
优先使用封装好的自动脚本（推荐，包含所有优化逻辑）：
```bash
./liepin_search.sh -k "<关键词>" -n <数量> [--city <城市>] [--experience <工作年限>] [--degree <学历>] -o <生成的文件名>
```

**示例：**
```bash
./liepin_search.sh -k "Java开发" -n 200 --city "上海" --experience "3-5年" --degree "本科" -o 猎聘_上海_3-5年_本科_Java开发_候选人_200位_20260415.csv
```

也可以直接执行opencli命令：
```bash
opencli liepin search "<关键词>" --limit <数量> [--city <城市>] [--experience <工作年限>] [--degree <学历>] -f csv > <文件名>.csv
```

**⚠️ 重要提示：** 不要在命令中添加`--headed`、`--persistent`、`--browser`等参数，这些参数opencli不支持，浏览器路径和启动模式由opencli daemon自动管理，只要Chrome已打开并连接扩展即可正常工作。

### 步骤 3: 处理 CSV 编码（Excel 兼容）

猎聘返回的 CSV 默认是 UTF-8 编码，但 Excel 打开可能乱码。**必须**添加 UTF-8 BOM 头：

```bash
python3 << 'EOF'
import codecs
filename = 'ai_product_managers.csv'  # 替换为实际文件名
with open(filename, 'rb') as f:
    content = f.read()
# 跳过已存在的 BOM 并添加 UTF-8 BOM
if content.startswith(codecs.BOM_UTF8):
    content = content[len(codecs.BOM_UTF8):]
with open(filename, 'wb') as f:
    f.write(codecs.BOM_UTF8)
    f.write(content)
print('已添加 UTF-8 BOM，Excel 可正常打开')
EOF
```

或者更简单的一行命令（如果系统支持）：
```bash
echo -ne '\xef\xbb\xbf' | cat - ai_product_managers.csv > temp.csv && mv temp.csv ai_product_managers.csv
```

## 常见问题与解决方案（坑点提炼）

### 坑点 1: opencli doctor 显示 extension not connected
**问题：** Extension 未连接
**解决方案：**
1. 确认 Chrome 浏览器已打开
2. 确认 opencli Browser Bridge 扩展已安装
3. 访问 `chrome://extensions/` 检查扩展是否启用
4. 重启 Chrome 浏览器

### 坑点 2: 命令执行后无输出或文件为空
**问题：** CSV 文件生成但内容为空或只有 header
**解决方案：**
1. 确认已在 Chrome 中登录 h.liepin.com
2. 检查账号是否有人才库搜索权限
3. 尝试在浏览器中手动搜索确认有结果
4. 增加 `--capture_seconds` 参数（默认 20，可设为 30-60）

### 坑点 3: Excel 打开 CSV 乱码
**问题：** 中文显示为乱码
**解决方案：**
- **必须** 添加 UTF-8 BOM 头，见"步骤 3"
- 不要用记事本打开后另存为，会破坏格式

### 坑点 4: 抓取数量不足或被风控
**问题：** 返回条数少于 --limit 指定值，或提示请求过快
**解决方案：**
1. 增加 `--page_delay` 参数（默认 2 秒，可设为 3-5 秒）
2. 分批抓取（每次 50-100 条，间隔一段时间）
3. 使用 `--pagination browser` 模式（默认就是 browser）

### 坑点 5: 提示 "No adapter found for liepin"
**问题：** opencli 版本太旧，没有 liepin 适配器
**解决方案：**
```bash
npm update -g @jackwener/opencli
```

### 坑点 6: 命令执行超时
**问题：** 命令长时间无响应后超时
**解决方案：**
1. 检查网络连接
2. 减少 `--limit` 值（先试 20 条）
3. 增加环境变量超时时间：
```bash
OPENCLI_BROWSER_COMMAND_TIMEOUT=120 opencli liepin search ...
```

### 坑点 7: 提示 "unknown option '--headed'"
**问题：** 执行命令时提示`--headed`、`--persistent`、`--browser`等参数是未知选项
**原因：** 这些参数不是opencli liepin search命令的参数，也不是opencli全局支持的参数，opencli会自动使用已打开的Chrome浏览器
**解决方案：**
1. 从命令中移除这些不支持的参数
2. 确保Chrome浏览器已打开，且已登录猎聘招聘端（h.liepin.com）
3. 确保opencli Browser Bridge扩展已连接（运行`opencli doctor`验证）

## 注意事项

1. **账号安全**：不要频繁大量抓取，避免账号被风控
2. **数据隐私**：候选人信息属于敏感数据，妥善保管
3. **时间间隔**：多次搜索之间建议间隔数分钟
4. **权限确认**：确保猎聘账号有人才库访问权限
